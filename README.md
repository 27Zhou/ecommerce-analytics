# 社交电商数据分析系统

[![Python](https://img.shields.io/badge/Python-3.10+-blue.svg)](https://python.org)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

> 从 10 万条用户行为数据中挖掘价值，找到业务增长的核心答案

## 项目简介

本项目是一套面向社交电商场景的数据分析系统，基于 **10 万条真实用户行为数据**（覆盖用户画像、商品属性、互动行为、购买标签 32 个核心字段），帮助商家解决以下痛点：

- 画像不清 → 无法精准营销
- 商品盲目 → 不知哪些商品受欢迎
- 发券随意 → 优惠券转化率低
- 无法预测 → 错失销售机会

系统提供 **6 大分析模块 + 25 张可视化图表 + 3 类机器学习模型**，支持一键导出 Excel 报告，实现数据驱动的精细化运营。

## 核心功能

| 模块 | 功能 | 技术方法 |
|------|------|----------|
| 数据概览 | 核心指标看板 | Pandas 聚合统计 |
| 用户画像 | RFM 分层 + 价值分层 | 加权评分 |
| 商品分析 | 类目热度、价格区间、折扣曲线、热度指数 | 多维加权评分 |
| 优惠券策略 | 使用率统计、A/B 测试矩阵 | 交叉分析 + 热力图 |
| 消费倾向预测 | 用户购买概率预测 | LR + RF + XGBoost + Stacking |
| 社交影响力 | KOL 识别、影响力评分 | 多维度加权评分模型 |

## 快速开始

### 环境要求

- Python 3.10 或更高版本

### 安装步骤

```bash
# 克隆项目
git clone https://github.com/27Zhou/ecommerce-analytics.git
cd ecommerce-analytics

# 安装依赖
pip install -r requirements.txt

# 运行程序
python main.py
```

## 核心算法说明

### 1. 数据清洗与预处理

项目基于 10 万条用户行为数据进行清洗：

- **去重**：按 `user_id` + `item_id` 去除重复记录
- **缺失值处理**：数值列用中位数填充，类别列用众数填充
- **异常值标记**：基于 3σ 原则（均值 ± 3 倍标准差）标记刷单嫌疑数据

### 2. RFM 用户分层模型

RFM 模型通过三个维度综合评估用户价值：

| 维度 | 含义 | 评分规则 |
|------|------|----------|
| R (Recency) | 最近购买时间 | 间隔≤3天=1分，≤14天=2分，>14天=3分 |
| F (Frequency) | 购买频率 | 购买≥15次=1分，≥5次=2分，<5次=3分 |
| M (Monetary) | 消费金额 | 消费≥5000元=1分，≥1500元=2分，<1500元=3分 |

**分层规则**：总分≤4分为高价值用户，5-7分为一般用户，8-9分为流失风险用户。

### 3. 商品热度指数

商品热度指数综合评估商品的受欢迎程度：

| 行为 | 权重 | 设计理由 |
|------|------|----------|
| 购买 | 0.35 | 真实转化，含金量最高 |
| 评论 | 0.20 | 深度互动，反映用户投入度 |
| 收藏 | 0.20 | 高意向行为，表明购买意愿 |
| 点赞 | 0.15 | 轻度互动，门槛较低 |
| 浏览 | 0.10 | 基础曝光，最易获取 |

**计算公式**：
```
热度指数 = 购买×0.35 + 评论×0.20 + 收藏×0.20 + 点赞×0.15 + 浏览×0.10
```

### 4. 社交影响力评分

社交影响力评分模型用于识别 KOL（关键意见领袖）：

| 指标 | 权重 | 设计理由 |
|------|------|----------|
| 分享次数 | 0.30 | 社交传播的核心行为 |
| 购买转化率 | 0.30 | 最终价值指标 |
| 互动率 | 0.25 | 内容质量指标 |
| 粉丝数 | 0.15 | 基础影响力（权重较低，防止刷粉干扰） |

**计算公式**：
```
影响力评分 = 分享×0.30 + 转化率×0.30 + 互动率×0.25 + 粉丝数×0.15
```

### 5. Stacking 集成学习

采用 Stacking 集成策略构建购买预测模型：

```
┌─────────────────────────────────────┐
│           基模型层 (Base Learners)   │
│  ┌─────────┐ ┌─────────┐ ┌────────┐│
│  │逻辑回归 │ │随机森林 │ │XGBoost ││
│  └────┬────┘ └────┬────┘ └────┬───┘│
│       └───────────┼────────────┘   │
├─────────────────────────────────────┤
│           元模型层 (Meta Learner)   │
│         ┌────────────────────┐      │
│         │   逻辑回归 (融合)   │      │
│         └────────────────────┘      │
└─────────────────────────────────────┘
```

### 6. SMOTE 过采样

处理购买/未购买样本不平衡问题，通过生成少数类合成样本提升召回率。

### 7. 时间序列划分

按 `register_days`（注册天数）升序排序，前 80% 作训练集，后 20% 作测试集，避免"用未来预测过去"的数据泄露问题。

## 可视化图表

系统共生成 25 张可视化图表，覆盖 6 大分析模块：

| 模块 | 图表 | 数量 |
|------|------|------|
| 数据概览 | 浏览量分布、类目销售占比、用户等级分布 | 3 |
| 用户画像 | 年龄分布、性别比例、RFM 分层、粉丝/关注分布、价值分层 | 5 |
| 商品分析 | 类目热度对比、价格区间转化率、折扣曲线、内容影响、标题情感、热度指数 | 6 |
| 优惠券策略 | 使用率饼图、A/B 测试热力图、未用券用户画像 | 3 |
| 消费倾向 | 模型对比、特征重要性、混淆矩阵、相关性热力图、错误样本分析、转化漏斗 | 6 |
| 社交影响力 | 粉丝分布、互动率 Top20、影响力评分分布、类目偏好、社交影响力指数 | 5 |
